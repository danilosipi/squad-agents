# Demanda

Criar os adaptadores Prisma dos contratos read/write do cap-base, começando pelos contratos usados pelos use cases já testados.

# Contexto

O cap-base já possui:

- schema Prisma inicial;
- contratos read/write;
- use cases de aplicação;
- testes unitários com fakes/in-memory;
- validação TypeScript sem erros;
- testes unitários passando.

# Objetivo

Definir qual fluxo de agentes deve ser executado antes de implementar os adaptadores Prisma.
